# 최근 업데이트 사항

## 핵심 버그 수정 (2024)

### 1. 🔥 주별 조회 시 스케줄 표시 안 되는 문제 수정 (핵심!)

**문제:**
- 월별 보기에서는 스케줄이 보이는데 주별 보기에서는 안 보이는 경우 발생
- 백엔드 API의 날짜 범위 조건이 너무 엄격함

**해결:**
- 백엔드: `WHERE s.start_time >= ? AND s.end_time <= ?` 
  → `WHERE (s.start_time <= ? AND s.end_time >= ?)` 로 변경
- 프론트엔드: 조회 범위를 현재 월 ±1.5개월로 확대

**효과:**
- 주별/일별 뷰에서도 모든 스케줄 정상 표시
- 캘린더 뷰 전환 시에도 데이터 누락 없음

---

### 2. 💅 시간 표시 글씨체 개선

**적용 내용:**
- SF Pro Display, Inter 폰트 적용
- font-weight: 600으로 가독성 향상
- letter-spacing 최적화
- tabular-nums로 숫자 정렬 개선

**효과:**
- 시간 표시가 더 선명하고 읽기 쉬움
- 다크모드에서도 가독성 유지

---

### 3. ⚡ 삭제 시 자동 새로고침

**변경 사항:**
- 스케줄 삭제 시 즉시 캘린더에서 제거
- 알바생 삭제 시 즉시 목록에서 제거
- 불필요한 페이지 새로고침 제거

**효과:**
- 즉각적인 UI 반응
- 더 나은 사용자 경험

---

### 4. 🎯 알바생 추가 시 자동 업데이트

**변경 사항:**
- 알바생 추가/수정/삭제 시 확인 alert 제거
- 자동으로 목록 업데이트
- 폼 자동 닫기

**효과:**
- 부드러운 작업 흐름
- 불필요한 클릭 제거

---

## 테스트 방법

### 1. 주별 스케줄 표시 테스트

```
1. 알바생 추가
2. 월별 뷰에서 스케줄 추가
3. 주별 뷰로 전환
4. ✅ 스케줄이 정상적으로 보여야 함
5. 일별 뷰로 전환
6. ✅ 스케줄이 정상적으로 보여야 함
```

### 2. 삭제 기능 테스트

```
1. 스케줄 클릭하여 삭제
2. ✅ 즉시 캘린더에서 사라져야 함 (새로고침 없이)
3. 알바생 삭제
4. ✅ 즉시 목록에서 사라져야 함 (새로고침 없이)
```

### 3. 알바생 추가 테스트

```
1. "+ 추가" 클릭
2. 정보 입력 후 "추가" 클릭
3. ✅ alert 없이 바로 목록에 추가되어야 함
4. ✅ 폼이 자동으로 닫혀야 함
```

### 4. 시간 표시 테스트

```
1. 주별 또는 일별 뷰로 이동
2. ✅ 왼쪽 시간 표시가 선명하고 이쁘게 보여야 함
3. 다크모드 전환
4. ✅ 시간 표시가 여전히 가독성 좋아야 함
```
