# 502 에러 해결 가이드

## 증상
```
알바생 목록 불러오기 실패: AxiosError: Request failed with status code 502
```

## 원인
502 Bad Gateway는 nginx(프론트엔드)가 백엔드 서버에 연결하지 못할 때 발생합니다.

## 해결 방법

### 1단계: 컨테이너 상태 확인

```bash
docker-compose ps
```

**예상 출력:**
```
NAME                  STATUS
parttime-backend      Up
parttime-frontend     Up
```

**만약 backend가 "Exit" 또는 "Restarting"이라면:**

```bash
# 백엔드 로그 확인
docker-compose logs backend

# 일반적인 원인:
# - 포트 충돌 (15000번 포트가 이미 사용 중)
# - 데이터베이스 초기화 실패
# - 의존성 설치 실패
```

### 2단계: 백엔드 연결 테스트

```bash
# 방법 1: 호스트에서 직접 테스트
curl http://localhost:15000/api/health

# 방법 2: 프론트엔드 컨테이너에서 테스트
docker exec parttime-frontend wget -qO- http://backend:15000/api/health
```

**정상 응답:**
```json
{"status":"ok","timestamp":"2024-01-01T00:00:00.000Z"}
```

### 3단계: 컨테이너 재시작

```bash
# 전체 재시작
docker-compose down
docker-compose up -d

# 백엔드만 재시작
docker-compose restart backend

# 로그 실시간 확인
docker-compose logs -f backend
```

### 4단계: 포트 충돌 확인

```bash
# 15000번 포트 사용 확인
sudo netstat -tlnp | grep 15000

# 또는
sudo ss -tlnp | grep 15000

# 다른 프로세스가 사용 중이라면 종료
sudo kill -9 [PID]
```

### 5단계: 완전 재빌드

캐시 문제일 수 있으므로 완전히 재빌드:

```bash
# 모든 컨테이너 및 볼륨 삭제
docker-compose down -v

# 이미지 삭제
docker rmi parttime-management-backend parttime-management-frontend

# 캐시 없이 재빌드
docker-compose build --no-cache

# 실행
docker-compose up -d

# 로그 확인
docker-compose logs -f
```

### 6단계: 데이터베이스 권한 확인

```bash
# data 디렉토리 권한 확인
ls -la data/

# 권한 수정 (필요시)
sudo chmod -R 777 data/
```

### 7단계: .env 파일 확인

```bash
# .env 파일이 있는지 확인
cat .env

# 없다면 생성
cat > .env << EOF
FRONTEND_URL=http://YOUR_SERVER_IP:3000
EOF

# 백엔드 재시작
docker-compose restart backend
```

## 빠른 해결 스크립트

진단 스크립트 실행:

```bash
chmod +x diagnose.sh
./diagnose.sh
```

## 일반적인 해결 순서

1. **가장 빠른 방법**: 재시작
   ```bash
   docker-compose restart backend
   ```

2. **조금 더 확실한 방법**: 전체 재시작
   ```bash
   docker-compose down
   docker-compose up -d
   ```

3. **완전한 방법**: 재빌드
   ```bash
   docker-compose down -v
   docker-compose build --no-cache
   docker-compose up -d
   ```

## 여전히 안 된다면

### 수동으로 백엔드 컨테이너 접속해서 확인

```bash
# 백엔드 컨테이너 접속
docker exec -it parttime-backend sh

# 내부에서 서버가 실행 중인지 확인
ps aux | grep node

# 포트 리스닝 확인
netstat -tlnp | grep 15000

# 수동으로 서버 실행 시도
cd /app
node server.js
```

### 로그에서 확인할 내용

```bash
docker-compose logs backend
```

**찾아야 할 메시지:**
- ✅ `🚀 서버가 포트 15000에서 실행 중입니다.`
- ✅ `✅ 데이터베이스 초기화 완료`
- ❌ `Error: listen EADDRINUSE` (포트 충돌)
- ❌ `Error: EACCES` (권한 문제)
- ❌ `Error: Cannot find module` (의존성 문제)

## 문제별 해결 방법

### "EADDRINUSE" 에러
```bash
# 포트 충돌 - 다른 프로세스 종료
sudo lsof -ti:15000 | xargs kill -9
docker-compose up -d
```

### "EACCES" 에러
```bash
# 권한 문제
sudo chmod -R 777 data/
docker-compose restart backend
```

### "Cannot find module" 에러
```bash
# 의존성 재설치
docker-compose build --no-cache backend
docker-compose up -d
```

### 네트워크 연결 안 됨
```bash
# Docker 네트워크 재생성
docker-compose down
docker network prune -f
docker-compose up -d
```

## 최종 확인

모든 것이 정상이라면:

```bash
# 1. 컨테이너 상태

# → 둘 다 "Up" 상태여야 함

# 2. 백엔드 헬스체크

# → {"status":"ok",...}

# 3. 브라우저에서 접속
# http://YOUR_SERVER_IP:3000
# → 알바생 관리 화면 정상 표시
```
